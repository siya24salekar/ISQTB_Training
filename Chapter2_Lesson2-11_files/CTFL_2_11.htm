Lesson 2-11

Re-testing and Regression Testing

Once a defect has been discovered and fixed, the software should be re-tested to check that the original Defect really has been successfully fixed. The term for this is ‘confirmation testing’, but a more widely used term is ‘re-testing’. You may also come across the term ‘debugging (the locating and fixing of defects)' or ‘defect fixing’, but these are considered to be a development activity, and not a testing activity.

A widely used form of testing today is ‘regression testing’. This type of testing is achieved by the repeated testing of a previously tested program that has been modified. The aim of this is to discover any defects introduced or uncovered as a result of the change(s) made to the software.

Often, the defects found during regression testing are not found in the functionality that has been modified, but in a different area of functionality that is related to the area of change. At some point a decision will have to be made as to how much regression testing should be performed, this is normally done by assessing the risk of what will happen if a defect is not found by the testing.

Test case design plays a major factor is regression testing, as typically regression tests are simply a selection of functional test cases that have been previously executed. So it’s important to ensure any test case design on a project at least considers that the test case should be repeatable. This way it can later be a candidate for a regression test. Due to the repeatable aspect of this type of testing, regression testing is a prime candidate for test automation.

Make sure you understand the following keywords associated with this lesson: 

Regression testing:

Testing of a previously tested program following modification to ensure that defects have not been introduced or uncovered in unchanged areas of the software, as a result of the changes made. It is performed when the software or its environment is changed.

Re-testing:

Testing that runs test cases that failed the last time they were run, in order to verify the success of corrective actions.
