Lesson 2-6

System Testing

System testing is testing that is aimed at the system or product as a whole. The scope of the system is defined by the development project. The main aim of System testing is to replicate how the product will be used by the end user. This is achieved by creating a test environment that has the characteristic of a real-world environment. By performing System testing this way, the risk of missing environment-specific failures can be significantly reduced. System testing is often performed by a dedicated systems tester or a system testing department. This is due to the fact that specific knowledge of hardware and software testing tools is often needed in order to perform the testing.

 System testing may include test cases based on a large variety of inputs, such as:

  • Risk

  • Requirements specifications

  • Business processes

  • Use cases

  • High-level descriptions of system behavior

System testing should be used to test against functional and non-functional requirements of the system. Ideally, these requirements should be well-defined in order to provide the system testers with enough information for them to provide meaningful results. For example; consider the following requirement: ‘the systems response time to a request should be within an acceptable amount of time.’ This does not provide enough information for a system test against this requirement because we don’t know what is an ‘acceptable’ amount of time. System testers may also find situations where there are incomplete requirements, or even no requirements at all.

Make sure you understand the following keywords associated with this lesson: 

System testing:

The process of testing an integrated system to verify that it meets specified requirements.

Test environment:

An environment containing hardware, instrumentation, simulators, software tools, and other support elements needed to conduct a test.
