Lesson 4-2

Categories of Test Design Techniques

The reason why we use different test design techniques is to identify test conditions, test cases, and test data for each area of test work. A common use of testing terminology is to classify a test technique as either black-box or white-box. Black-box test design techniques (often referred to as specification-based techniques) are a way to derive and select test conditions, test cases, or test data based on an analysis of the test basis documentation. This method includes both functional and non-functional testing.

By its very nature black-box testing does not actively use any information regarding the internal structure of the component or system to be tested. So this essentially means the tester will test the component or system without knowing what is going on in the background.

White-box test design techniques (often referred to as structural or structure-based techniques) are based on an analysis of the structure of the component or system. Black-box and white-box testing are used in conjunction with the knowledge and experience of developers, testers and users to work out what should be tested.

While some techniques can easily be associated with a single category; others include attributes of more than one category. For the purposes of this training course, specification-based test design techniques are referred to as black-box techniques and structure-based test design techniques are referred to as white-box techniques. In addition to this, experience-based test design techniques are also covered.

Some common characteristics of black-box test design techniques include:

  • Models, either formal or informal, are used for the specification of the problem to be solved, the software or its components

  • Test cases can be derived systematically from these models


    Common characteristics of white-box test design techniques include:

  • Information about how the software is constructed is used to derive the test cases (e.g., code and detailed design information)

  • The extent of coverage of the software can be measured for existing test cases, and further test cases can be derived systematically to increase coverage


    Common characteristics of experience-based test design techniques include:

  • The knowledge and experience of people are used to derive the test cases

  • The knowledge of testers, developers, users and other stakeholders about the software, its
    usage and its environment is one source of information

  • Knowledge about likely defects and their distribution is another source of information

Make sure you understand the following keywords associated with this lesson: 

Black box test design technique:

Procedure to derive and/or select test cases based on an analysis of the specification, either functional or non-functional, of a component or system without reference to its internal structure.

Experience-based test design technique:

Procedure to derive and/or select test cases based on the tester’s experience, knowledge and intuition.

Test design technique:

Procedure used to derive and/or select test cases.
