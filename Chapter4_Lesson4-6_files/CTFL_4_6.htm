Lesson 4-6

State Transition Testing

A software program or system can sometimes behave differently depending on its current condition or previous history. This is termed it's 'state'.

We can display this information by using a state transition diagram. A great advantage of using a state transition diagram, especially to a tester; is that it displays the software in terms of its states, transitions between states, the inputs or events that trigger state changes (transitions) and also the actions which can result from those transitions.

Typically, the states of the object under test are separate, identifiable and have a fixed amount of states. Testers can create a state table that shows the relationship between all of the identifiable states and inputs. It should then be possible to see any transitions that are invalid. The state table is a good starting point before any test cases are written. Once a state table has been created, test cases can be designed to cover a typical sequence of states, to cover every state, to exercise every transition, to exercise specific sequences of transitions or to test invalid transitions.

The concept of state transition testing is widely used within many areas of software development, but more commonly found in the embedded software industry and technical automation areas of software development. It can also be used for modeling a business object that has specific states or testing user interface screen-dialogue flows, such as internet applications or business scenarios.

The big advantage of a decision table is that it creates combinations of conditions that otherwise may not get tested. It may be applied to any situation where the action of the software depends on multiple logical decisions.

A simple example of how a decision table works:

  Applied For Rejected Processing In Use Inactive Closed
Applied For n y y y n n
Rejected y n n y n n
Processing n y n y n n
In Use n n n n y y
Inactive n n n y n y
Closed n n n y n n

The example state table shown above is designed to show the possible states of a medical center’s patient record system. You can clearly see which state transitions are valid and invalid. Valid state transitions are indicated by a 'Y' in the column where the ‘from’ and ‘to’ states intersect. For invalid state transitions, an 'N' is displayed in the column where the ‘from’ and ‘to’ states intersect.

Make sure you understand the following keywords associated with this lesson: 

State transition:

A transition between two states of a component or system.

State transition testing:

A black box test design technique in which test cases are designed to execute valid and invalid state transitions.
