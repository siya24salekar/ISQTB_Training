Lesson 4-8

Structure-based or White-box Techniques

Structure-based or white-box testing is based on testing with the knowledge of the internal workings of the software/system.

The following are examples of where structure-based testing is performed:

Component level:
The structure of a software component, i.e. statements, decisions, branches or even distinct paths

 

Integration level:

The structure may be a call tree (a diagram in which modules call other modules)


System level:

The structure may be a menu structure, business process or web page structure



Statement Testing and Coverage

In terms of component testing, statement coverage can be thought of as the assessment of the percentage of executable statements that have been exercised by a test case suite. Statement testing works by deriving test cases to execute specific statements, typically to increase statement coverage.

Statement coverage itself is calculated by observing the number of executable statements covered by test cases, which is then divided by the number of all executable statements in the code under test.

Statement Coverage = Number of executable statements executed
Total number of executable statements * 100 %


Let's look at the code sample below, and try to determine how many tests are required to achieve 100% statement coverage:

Read X
Read Y
IF X<5 THEN
Print "X is a negative value"
ENDIF
IF Y<8 THEN
Print "Y is a negative value"
END

So we can see that there are two 'executable' statements there, which are 'IF X<5 THEN' and 'IF Y<8 THEN'.

Our test case will need to exercise both of these statements if we are to achieve 100% statement coverage. The test case below uses two values of 'X' and 'Y' which will exercise both 'IF' statements resulting in both messages being printed:

Test case: 'X=4' and 'Y=3'

Make sure you understand the following keywords associated with this lesson: 

Code coverage:

An analysis method that determines which parts of the software have been executed (covered) by the test suite and which parts have not been executed, e.g. statement coverage, decision coverage or condition coverage.

Statement testing:

A white box test design technique in which test cases are designed to execute statements.

Statement coverage:

The percentage of executable statements that have been exercised by a test suite.
