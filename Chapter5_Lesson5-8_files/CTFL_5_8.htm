Lesson 5-8

Configuration Management

Configuration management is used in virtually all professional software developments today. It is used to establish and maintain the integrity of the products of the software or system throughout the project life-cycle. These products may consist of software components, data or documentation for example.

With regards to testing, configuration management may involve any or all of the following:

a) All items of testware are identified, version controlled, tracked for changes, related to each other and related to development items (test objects) so that traceability can be maintained throughout the test process.

b) All identified documents and software items are referenced unambiguously in test documentation

c) For the individual tester, configuration management can be used to assist with uniquely identifying items such as the tested item, test documents, the tests and the test harness(es). This information is especially useful when trying to reproduce specific environments and scenarios.

During the test planning stage of a project, any configuration management procedures and infrastructure (i.e tools) should be chosen, documented and implemented to ensure the testers can easily utilize the many benefits that configuration management offers.

Make sure you understand the following keywords associated with this lesson: 

Configuration management: A discipline applying technical and administrative direction and surveillance to: identify and document the functional and physical characteristics of a configuration item, control changes to those characteristics, record and report change processing and implementation status, and verify compliance with specified requirements.

Configuration control:

An element of configuration management, consisting of the evaluation, co-ordination, approval or disapproval, and implementation of changes to configuration items after formal establishment of their configuration identification.
