Lesson 6-3

Tool Support for Management of Testing and Tests

Management tools can be considered to be apply to all test related activities spread over an entire software life cycle.

Test Management Tools

These tools provide interfaces for tracking defects, executing tests and managing requirements. Also provisions for quantitative analysis and reporting of the each test object should be included. These tools also support tracking the test objects to requirement documentation (specifications). Some test management tools may have built-in version control functionality, or the capability to link to an external one.

Requirements Management Tools

These tools will hold requirement statements, store requirement attributes (like priority etc.), provide unique identifiers and allow tracing of the requirements to individual tests cases. These tools can provide useful information such as highlighting requirement inconsistencies, or even missing requirements.

Incident Management Tools (or Defect Tracking Tools)

These tools manage and store reports based on incidents (incident reports). The incidents can be defects, failures, change requests or perceived problems and anomalies. These tools can help in managing the life cycle of incidents, and often have the ability to offer statistical analysis.

Configuration Management Tools

Technically, these tools are not test tools, but they serve a necessary purpose in relation to testing, such as storage and version management of testware and other related items. These tools come into their own, especially when configuring more than one test environment that implements multiple operating system versions, compilers, browsers and such like.

Tool Support for Static Testing

These tools allow for an effective and essentially low cost method of finding more defects earlier on in the
development life-cycle.

Review Tools

These tools can aid review processes, review guidelines, checklists. They are used to store and communicate information, such as review comments, defect reports and review effort. They often used to great advantage by using them for online reviews, which are of interest for review teams that are globally dispersed.

Static Analysis Tools (D)

These tools assist developers and testers alike to locate defects before any dynamic testing begins. By providing support for enforcing coding standards (such as secure coding), analysis of structures and dependencies. They can also greatly assist in helping, planning or assessing risks analysis by providing metrics for the code, which in turn shows us the 'complexity'.

Modeling Tools (D)

These tools are typically used to validate software models. An example of a software model is the 'physical data model' or PDM. These tools often assist with generating test cases, which are based on the models.

Make sure you understand the following keywords associated with this lesson: 

Test management tool:

A tool that provides support to the test management and control part of a test process. It often has several capabilities, such as testware management, scheduling of tests, the logging of results, progress tracking, incident management and test reporting.

Requirements management tool:

A tool that supports the recording of requirements, requirements attributes (e.g. priority, knowledge responsible) and annotation, and facilitates traceability through layers of requirements and requirements change management. Some requirements management tools also provide facilities for static analysis, such as consistency checking and violations to pre-defined requirements rules.

Incident management tool:

A tool that facilitates the recording and status tracking of incidents. They often have workflow-oriented facilities to track and control the allocation, correction and re-testing of incidents and provide reporting facilities.

Configuration management tool:

A tool that provides support for the identification and control of configuration items, their status over changes and versions, and the release of baselines consisting of configuration items.

Review tool:

A tool that provides support to the review process. Typical features include review planning and tracking support, communication support, collaborative reviews and a repository for collecting and reporting of metrics.

Static analyzer:

A tool that carries out static analysis.

Modeling tool:

A tool that supports the creation, amendment and verification of models of the software or system.
