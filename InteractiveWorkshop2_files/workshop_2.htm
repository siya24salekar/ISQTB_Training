ISEB/ISTQB Workshop 2 - Equivalence Partitioning

1. Valid Equivalence Partitions

The 'Employee ID' field is designed to allow a value between 1 and 1000, and the 'Hours Worked' field allows a value between 1 and 99999.

 

Enter values into both boxes and click 'Enter'

 

The correct valid equivalence partition values are:

Employee ID - any value between 1 and 1000

Hours Worked - any value between 1 and 99999

2. Invalid Equivalence Partitions

The 'Employee ID' field is designed to allow a value between 1 and 1000, and the 'Hours Worked' field allows a value between 1 and 99999.

 

Enter values into both boxes and click 'Enter'

 

The correct invalid equivalence partition values are:

Employee ID - any value lower than 1

Employee ID - any value higher than 1000

Hours Worked - any value lower than 1

Hours Worked - any value higher than 99999

3. Valid Equivalence Partitions

The 'Employee ID' field is designed to allow a value between 500 and 1000. Va and the 'Hours Worked' field allows a value between 100 and 1000.

 

Enter values into both boxes and click 'Enter'

 

The correct valid equivalence partition values are:

Employee ID - any value between 500 and 1000

Hours Worked - any value between 100 and 1000

4. Invalid Equivalence Partitions

The 'Employee ID' field is designed to allow a value between 500 and 1000, and the 'Hours Worked' field allows a value between 100 and 1000.

 

Enter values into both boxes and click 'Enter'

 

The correct invalid equivalence partition values are:

Employee ID - any value lower than 500

Employee ID - any value higher than 1000

Hours Worked - any value lower than 100

Hours Worked - any value higher than 1000

5. Valid Equivalence Partitions - Values that exercise different code

The 'Employee ID' field is designed to allow a value between 500 and 1000, and values over 800 here will automatically print a report. The 'Hours Worked' field allows a value between 100 and 1000, and and values over 600 here will automatically send an email to the Administrator.

Enter values from the equivalence partitions that exercise the 'print' functionality and also the 'email' functionality.

 

Enter values into both boxes and click 'Enter'

 

The correct equivalence partition values are:

Employee ID - any value between 801 and 1000

Hours Worked - any value between 601 and 1000

6. All Equivalence Partitions - Values that exercise different code

The 'Employee ID' field is designed to allow a value between 400 and 800, and values over 500 here will automatically print a report. The 'Hours Worked' field allows a value between 100 and 500, and and values over 350 here will automatically send an email to the Administrator.

Enter values from all equivalence partitions, including valid and invalid partitions.

 

Enter values into both boxes and click 'Enter'

 

The correct equivalence partition values are:

Employee ID - any value lower than 400

Employee ID - any value between 400 and 500

Employee ID - any value between 501 and 800

Employee ID - any value higher than 800

Hours Worked - any value lower than 100

Hours Worked - any value between 100 and 350

Hours Worked - any value between 351 and 500

Hours Worked - any value higher than 500

The correct invalid upper boundary values are '5 digits' for the Employee ID and '13 characters' for the Hours Worked.
