Practice Quiz 10

1) Testing gives confidence in the quality of the software if it finds what? Show Answer

The correct Answer is 'C'
a) Severe defects
b) Major defects
c) No defects
d) Lots of minor defects

2) Subsequent what, by a tester ensures that the fix does indeed resolve the failure? Show Answer

The correct Answer is 'A'
a) Re-testing
b) Unit testing
c) Functional testing
d) Regression testing

3) Instead of exhaustive testing, what should be used? Show Answer

The correct Answer is 'C'
a) Regression testing
b) Usability testing
c) Risk analysis
d) Static analysis

4) Finding and fixing defects does not help if the system built does not fulfill the users what? Show Answer

The correct Answer is 'C'
a) Office requirements
b) Environment
c) Expectations
d) System

5) In order to control testing, testing activities should be ______ during the project. Show Answer

The correct Answer is 'C'
a) Postponed
b) Customized
c) Monitored
d) Maintained

6) Test ______ takes into account the feedback from monitoring and control activities. Show Answer

The correct Answer is 'C'
a) Test results
b) Process
c) Planning
d) Control

7) What can be used to help focus effort and provide additional benefits? Show Answer

The correct Answer is 'B'
a) Regression testing
b) Independence
c) Use of contractors
d) Testing

8) Which of the following is not a valid level of independence? Show Answer

The correct Answer is 'B'
a) Tests designed by another person(s)
b) Tests designed by the person(s) who tested the software under test
c) Tests designed by a person(s) from a different organizational group
d) Tests designed by a person(s) from a different organization or company

9) Which of the following is not considered to be under the ISTQB code of ethics? Show Answer

The correct Answer is 'C'
a) Judgement
b) Profession
c) Collaboration
d) Self

10) What does CMMI represent? Show Answer

The correct Answer is 'B'
a) Component Maturity Model Integration
b) Capability Maturity Model Integration
c) Capability Maternity Model Integration
d) Company Maturity Model Integration

11) The Test Basis for Unit testing consist of all, but which of the following? Show Answer

The correct Answer is 'C'
a) Component requirements
 b) Detailed design
c) Test suites
 d) Code

12) Typically, component testing occurs with access to what? Show Answer

The correct Answer is 'A'
a) The code
b) The quality metrics
c) The test plan
d) The Team leader

13) A test-first approach is highly what? Show Answer

The correct Answer is 'C'
a) Recommended
b) Desirable
c) Iterative
d) Repetitive

14) Which of the following is not part of the test basis for System testing? Show Answer

The correct Answer is 'C'
a) Use cases
b) Functional specification
c) Acceptance tests
d) Risk analysis reports

15) Development testing may include all but which of the following: Show Answer

The correct Answer is 'C'
a) Component Testing
b) Integration Testing
c) Acceptance Testing
d) System Testing

16) Which of the following is what a human can make? Show Answer

The correct Answer is 'C'
a) A bug
b) A defect
c) An error
d) A fault

17) Confirming that defects have been fixed, is also known as what? Show Answer

The correct Answer is 'B'
a) User Acceptance testing
 b) Confirmation testing
c) Acceptance testing
d) Regression testing

18) White-box testing may be performed at which level? Show Answer

The correct Answer is 'B'
a) Requirements review
b) All levels
c) Code Development
d) Functional testing

19) Which test suites are run many times and generally evolve slowly? Show Answer

The correct Answer is 'C'
a) Performance tests
b) Load tests
c) Regression tests
d) Usability tests

20) Maintenance testing for the retirement of a system may include the testing of what? Show Answer

The correct Answer is 'B'
a) Performance testing
b) Data migration
c) Documentation testing
 d) Acceptance testing

21) Defects detected during reviews early in the life cycle are often _____ to remove. Show Answer

The correct Answer is 'C'
a) More expensive
b) Pointless
c) Much cheaper
d) Harder

22) Which of the following is not a valid formal review activity? Show Answer

The correct Answer is 'D'
a) Planning
b) Individual preparation
c) Follow-up
d) Checking on test criteria

23) Which of the following is not a characteristic of an Walkthrough review? Show Answer

The correct Answer is 'D'
a) Meeting led by author
 b) May take the form of scenarios
c) Open-ended sessions
d) Inexpensive way to get some benefit

24) Which of the following is not a success factor for a review? Show Answer

The correct Answer is 'C'
a) Each review has clear predefined objectives
b) The right people for the review objectives are involved
c) Management supports a bad review process
d) Defects found are welcomed and expressed objectively

25) Which of the following is not a value of static analysis? Show Answer

The correct Answer is 'C'
a) Identification of defects not easily found by dynamic testing
b) Detecting dependencies and inconsistencies in software models such as links
c) Removing dependencies and inconsistencies in software models
d) Improved maintainability of code and design

26) During what are the test cases and test data are created and specified? Show Answer

The correct Answer is 'C'
a) Test analysis
b) Test execution
c) Test design
d) Test planning

27) Detailed what are helpful in determining the interesting boundaries? Show Answer

The correct Answer is 'A'
a) Test cases
b) Requirements
c) Inputs
d) Specifications

28) Structure-based or white-box testing is seen in all but which of the following? Show Answer

The correct Answer is 'B'
a) Component level
b) All levels
c) Integration level
d) System level

29) Decision testing is a form of what? Show Answer

The correct Answer is 'C'
a) Decision testing
b) Boundary value analysis
c) Control flow testing
d) Statement testing

30) Exploratory testing involves all but which of the following? Show Answer

The correct Answer is 'B'
a) Test design
b) Teaching
c) Test execution
d) Test logging

31) Testing tasks can often done by people in all but which of the following roles? Show Answer

The correct Answer is 'A'
a) Human resources manager
 b) Project manager
c) Quality manager
 d) Business expert

32) Typically who plans, monitors and controls the testing activities and tasks? Show Answer

The correct Answer is 'A'
a) Test leader
b) Development manager
c) Quality assurance manager
d) Business analyst

33) Test planning may be documented in a what? Show Answer

The correct Answer is 'D'
a) Project plan
b) Business plan
c) Test specification
d) Master test plan

34) Feedback from test activities is used to recognize changing what? Show Answer

The correct Answer is 'D'
a) Times
b) Test plans
c) Requirements
d) Risks

35) Typical entry criteria may cover all but which of the following? Show Answer

The correct Answer is 'C'
a) Test tool readiness in the test environment
b) Testable code availability
c) Test environment setup and resources
d) Test data availability

36) Typically exit criteria may cover all but which of the following? Show Answer

The correct Answer is 'C'
a) Thoroughness measures, such as coverage of code, functionality or risk
b) Residual risks, such as defects not fixed
c) Estimates of defect severity or testability measures
d) Schedules such as those based on time to market

37) What is the implementation of the test strategy for a specific project? Show Answer

The correct Answer is 'B'
a) The test plan
b) The test approach
c) The test specification
d) The test cases

38) Typical test approaches include all but which of the following? Show Answer

The correct Answer is 'B'
a) Analytical approaches
b) Dynamic and hourglass approaches
c) Model-based approaches
 d) Methodical approaches

39) Common test metrics include all but which of the following? Show Answer

The correct Answer is 'C'
a) Test case execution
b) Test coverage of requirements
c) Percentage of failed tests cases when reviewed
d) Dates of development milestones

40) All but which of the following is related to project risk? Show Answer

The correct Answer is 'A'
a) Poor software characteristics
b) Organizational factors
 c) Technical issues
d) Supplier issues
