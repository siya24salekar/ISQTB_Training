Practice Quiz 11

1) The quality of the software system increases when what are fixed? Show Answer

The correct Answer is 'C'
a) Salaries
b) Release dates
c) Defects
d) Versioning

2) Lessons should be learned from what? Show Answer

The correct Answer is 'A'
a) Previous projects
b) Future projects
c) Rival companies
d) Interns

3) Testing can show that what are present? Show Answer

The correct Answer is 'C'
a) Developers
b) Lines of code
c) Defects
d) Test cases

4) The fundamental test process consists of all but which following activities? Show Answer

The correct Answer is 'C'
a) Implementation and execution
b) Test closure activities
 c) Planning and quality
d) Analysis and design

5) Test control is the ongoing activity of comparing actual progress against the what? Show Answer

The correct Answer is 'C'
a) Process
b) Design
c) Plan
d) Idea

6) Testing should be done differently in different what? Show Answer

The correct Answer is 'C'
a) Departments
b) Areas
c) Contexts
d) Teams

7) Even if no 'what' are found, it is not a proof of correctness. Show Answer

The correct Answer is 'B'
a) Test results
b) Defects
c) Failures
d) Faults

8) How many testing principles are there? Show Answer

The correct Answer is 'B'
a) 3
b) 7
c) 8
d) 5

9) What does COTS represent? Show Answer

The correct Answer is 'C'
a) Commercially On The Shelves
b) Computing Out The Shop
c) Commercial Off The Shelf
d) Completely On The Shelf

10) What does CMMI represent? Show Answer

The correct Answer is 'B'
a) Complex Maturity Model Integration
b) Capability Maturity Model Integration
c) Capability Maternity Model Internship
d) Company Maintenance Model Integration

11) Typical test objects for Unit testing consist of all, but which of the following? Show Answer

The correct Answer is 'C'
a) Components
b) Programs
c) Documentation
d) Data conversion / migration programs

12) Which of the following is not a typical test object for Integration testing? Show Answer

The correct Answer is 'A'
a) The test plan
b) Sub-systems database implementation
c) Infrastructure
d) Interfaces

13) Which testing is often the responsibility of the customers or users of a system? Show Answer

The correct Answer is 'C'
a) Regression testing
b) White-box testing
c) Acceptance testing
d) Functional testing

14) A large-scale system integration test may come after a what? Show Answer

The correct Answer is 'C'
a) Regression test
b) Functional test
c) Acceptance test
d) System test

15) Component testing is also known as all, but which of the following? Show Answer

The correct Answer is 'C'
a) Unit testing
b) Module testing
c) Black-box testing
d) Program testing

16) A ______ of the software may be developed and/or used in structural testing. Show Answer

The correct Answer is 'C'
a) Review
b) Overview
c) Model
d) Plan

17) Structural techniques are best used after which techniques? Show Answer

The correct Answer is 'B'
a) Functional-based
b) Specification-based
c) Test-based
 d) Review-based

18) Migration testing is also known as what? Show Answer

The correct Answer is 'B'
a) Usability testing
b) Conversion testing
c) Confirmation testing
d) Maintenance testing

19) Benefits of reviews include all but which of the following? Show Answer

The correct Answer is 'C'
a) Early defect detection and correction
b) Fewer defects
c) Development productivity decrease
d) Decreased communication

20) Which of the following is not a valid formal review activity? Show Answer

The correct Answer is 'B'
a) Checking entry criteria
 b) Examination/evaluation/recording of attendees comments
c) Follow-up
d) Individual preparation

21) Which of the following is not a characteristic of an Informal review? Show Answer

The correct Answer is 'C'
a) Results may be documented
 b) Varies in usefulness depending on the reviewers
c) Ideally led by trained moderator
d) Inexpensive way to get some benefit

22) Which of the following is not a characteristic of an Technical review? Show Answer

The correct Answer is 'D'
a) May be performed as a peer review
b) Pre-meeting preparation by reviewers
c) Optional use of checklists
d) Ideally led by the author

23) Which testing may be based on the architecture of the system? Show Answer

The correct Answer is 'D'
a) Functional testing
b) Black-box testing
c) Ad-hoc testing
 d) Structural testing

24) Which of the following is not a value of static analysis? Show Answer

The correct Answer is 'C'
a) Identification of defects not easily found by dynamic testing
b) Detecting dependencies and inconsistencies in software models such as links
c) Prevention of developers leaving, if lessons are learned in development
d) Improved maintainability of code and design

25) Static analysis tools are typically used by who? Show Answer

The correct Answer is 'C'
a) Testers
b) Managers
c) Developers
d) Customers

26) A test case consists of all but which of the following? Show Answer

The correct Answer is 'C'
a) Set of input values
b) Execution preconditions
c) Actual results
 d) Execution postconditions

27) Which of the following is not relate to a test execution schedule? Show Answer

The correct Answer is 'A'
a) Functional tests
b) Regression tests
c) Prioritization
 d) Technical and logical dependencies

28) What are typically used to test requirements that contain logical conditions? Show Answer

The correct Answer is 'B'
a) Test tables
b) Decision tables
c) Times tables
d) Boundary tables

29) What shows the relationship between the states and inputs? Show Answer

The correct Answer is 'C'
a) Cross-reference
b) State table
c) Decision table
d) Test table

30) Structure-based or white-box testing is seen in all but which of the following? Show Answer

The correct Answer is 'B'
a) Component level
b) Functional level
c) Integration level
d) System level

31) Which of the following is not a value of static analysis? Show Answer

The correct Answer is 'A'
a) Late warning about suspicious aspects of the code
b) Identification of defects not easily found by dynamic testing
c) Detecting dependencies and inconsistencies in software models such as links
d) Improved maintainability of code and design

32) Testing tasks can often be done by people in all but which of the following roles? Show Answer

The correct Answer is 'A'
a) Building maintenance worker
b) Project manager
c) Quality manager
 d) Business expert

33) Sometimes the test leader is called a what? Show Answer

The correct Answer is 'D'
a) A project worker
b) A project leader
c) A test planner
 d) A test coordinator

34) Which of the following is not a typical test leader task? Show Answer

The correct Answer is 'D'
a) Decide what should be automated, to what degree, and how
b) Decide about the implementation of the test environment
c) Write test summary reports based on the information gathered during testing
d) Adapt planning based on estimated results

35) What is influenced by the test policy of the organization? Show Answer

The correct Answer is 'C'
a) Project schedule
b) Test specification
c) Test planning
 d) Requirements

36) All but which of the following are Test planning activities? Show Answer

The correct Answer is 'C'
a) Scheduling test implementation, execution and evaluation
b) Selecting metrics for monitoring and controlling test preparation
c) Scheduling test basis and design implementations
d) Assigning defects for the different activities defined

37) Typically exit criteria may cover all but which of the following? Show Answer

The correct Answer is 'B'
a) Thoroughness measures, such as coverage of code, functionality or risk
b) Residual tests, such as defects not found
c) Estimates of defect density or reliability measures
d) Schedules such as those based on time to market

38) Which of the following is a valid test estimation approach? Show Answer

The correct Answer is 'B'
a) The business-based approach
b) The metrics-based approach
c) The Black-box approach
 d) The White-box approach

39) What is the starting point for planning the test process? Show Answer

The correct Answer is 'C'
a) The test plan
b) The test specification
c) The test approach
d) The test cases

40) Examples of test control actions include all but which of the following? Show Answer

The correct Answer is 'A'
a) Changing the test plan to reflect testers vacation time
b) Making decisions based on information from test monitoring
c) Re-prioritizing tests when an identified risk occurs
d) Setting an entry criterion requiring fixes to have been re-tested
