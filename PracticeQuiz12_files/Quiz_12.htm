Practice Quiz 12

1) Debugging is the development activity that finds, analyses and removes the cause of what? Show Answer

The correct Answer is 'C'
a) The culprit
b) The code
c) The failure
d) The bug

2) Which of the following is not a testing principle? Show Answer

The correct Answer is 'A'
a) Late testing
b) Testing shows presence of defects
c) Exhaustive testing is impossible
d) Defect clustering

3) Testing does not exist in what? Show Answer

The correct Answer is 'C'
a) Production
b) Projects
c) Isolation
d) Tandem

4) Which of the following is a valid Software life cycle process? Show Answer

The correct Answer is 'C'
a) AMC/T4S 12207
b) IEEE/IEC 72207
c) IEEE/IEC 12207
d) JJS-C/IEC 12207

5) Iterative-incremental development includes all but which of the following? Show Answer

The correct Answer is 'C'
a) Establishing requirements
 b) Designing
c) Done as a series of longer development cycles
d) Building and testing a system

6) Typical test objects for Unit testing consist of all, but which of the following? Show Answer

The correct Answer is 'C'
a) Components
b) Programs
c) Timescales
d) Data conversion / migration programs

7) A test-driven development approach is highly what? Show Answer

The correct Answer is 'B'
a) Recommended
b) Iterative
c) Desirable
d) Repetitive

8) A COTS software product may be what? when it is installed or integrated. Show Answer

The correct Answer is 'B'
a) Regression tested
b) Acceptance tested
c) Functional tested
d) System tested

9) Which testing of the usability of a component may be done during component testing? Show Answer

The correct Answer is 'C'
a) Regression testing
b) Functional testing
c) Acceptance testing
d) System testing

10) Looking for unintended changes is also known as what? Show Answer

The correct Answer is 'D'
a) User Acceptance testing
 b) Confirmation testing
c) Acceptance testing
d) Regression testing

11) What is the extent that a structure has been exercised by a test suite? Show Answer

The correct Answer is 'C'
a) Pass-rate
b) Percentage
c) Coverage
d) Boundary analysis

12) A review could be done entirely as a what? Show Answer

The correct Answer is 'A'
a) Manual activity
b) Development activity
c) Testing activity
d) Management activity

13) Benefits of reviews include all but which of the following? Show Answer

The correct Answer is 'C'
a) Development productivity improvements
b) Early defect detection and correction
c) Increased testing cost and time
d) Reduced development timescales

14) Which of the following is not a valid formal review planning activity? Show Answer

The correct Answer is 'C'
a) Defining the review criteria
b) Selecting the personnel
 c) Discussing or logging
d) Allocating roles

15) Which of the following is not a valid formal review activity? Show Answer

The correct Answer is 'C'
a) Checking entry criteria
 b) Follow-up
c) Re-test
d) Individual preparation

16) Which of the following is not a characteristic of an Technical review? Show Answer

The correct Answer is 'C'
a) May be performed as a peer review
b) Pre-meeting preparation by reviewers
c) Open-ended sessions
d) Optional use of checklists

17) What may offer some support for static analysis, including the calculation of metrics? Show Answer

The correct Answer is 'B'
a) Defect tracking systems
 b) Compilers
c) Test processes
 d) Virtual machines

18) Which of the following is the 'Standard for Software Test Documentation’? Show Answer

The correct Answer is 'B'
a) IEEE STD 928-0E7
b) IEEE STD 829-1998
c) T4S V9B
d) ISO 928-0E7

19) Boundary value analysis can be applied where? Show Answer

The correct Answer is 'C'
a) Functional testing
b) Structural testing
c) At all test levels
d) Regression testing

20) A state transition diagram displays all but which of the following? Show Answer

The correct Answer is 'B'
a) The states
b) The inputs which may result from the transitions
c) Transitions between states
d) Inputs or events that trigger state changes

21) Expected results should include all but which of the following? Show Answer

The correct Answer is 'C'
a) Include outputs.
b) Changes to data and states
c) A jungle refers to the thickest area of a rain forest
d) Any other consequences of the test

22) Sometimes the test leader is called a what? Show Answer

The correct Answer is 'D'
a) A test designer
b) A project leader
c) A test planner
 d) A test manager

23) The role of the test leader may be performed by all but which of the following? Show Answer

The correct Answer is 'D'
a) Project manager
b) Development manager
c) Quality assurance manager
d) Test automation expert

24) The outline of a test planning document is covered by which standard? Show Answer

The correct Answer is 'C'
a) IEEE Std V9B-T4S
b) IEEE Std 892-1998
c) IEEE Std 829-1998
d) ISO Std 829-1998

25) Test planning is influenced by all but which of the following? Show Answer

The correct Answer is 'C'
a) Scope of testing
b) Objectives
c) Restraints
d) Risks

26) All but which of the following are Test planning activities? Show Answer

The correct Answer is 'C'
a) Scheduling test analysis and design activities
b) Scheduling test implementation, execution and evaluation
c) Assigning defects for the different activities defined
d) Selecting metrics for monitoring and controlling test preparation

27) What can be used to define when to stop testing? Show Answer

The correct Answer is 'A'
a) Exit criteria
b) Entry criteria
c) Requirements
d) Test Specification

28) Typically exit criteria may cover all but which of the following? Show Answer

The correct Answer is 'B'
a) Estimates of defect density or reliability measures
b) Thoroughness of testing, such as coverage of code, functionality or reviews
c) Residual risks, such as defects not fixed
d) Schedules such as those based on time to market

29) What typically includes the decisions made based on the project’s goal and risk assessment? Show Answer

The correct Answer is 'C'
a) The test plan
b) The test specification
c) The test approach
d) The test cases

30) Typical test approaches include all but which of the following? Show Answer

The correct Answer is 'B'
a) Analytical approaches
 b) Modem-based approaches
c) Methodical approaches
d) Dynamic and heuristic approaches

31) Common test metrics include all but which of the following? Show Answer

The correct Answer is 'A'
a) Test coverage of design documentation
b) Percentage of work done in test case preparation
c) Test case execution
d) Dates of test milestones

32) All but which of the following is related to project risk? Show Answer

The correct Answer is 'A'
a) Failure-prone software delivered
b) Organizational factors
 c) Technical issues
d) Supplier issues

33) Which of the following is not an organizational factor relating to project risk? Show Answer

The correct Answer is 'D'
a) Skill, training and staff shortages
b) Personnel issues
c) Improper attitude toward or expectations of testing
d) Improper management attitude

34) All of the following are product risks except what? Show Answer

The correct Answer is 'D'
a) Failure-prone software delivered
b) Software that does not perform its intended functions
c) Poor software characteristics
d) Problems in defining the right requirements

35) Which of the following is not a testing principle? Show Answer

The correct Answer is 'C'
a) Exhaustive testing is impossible
b) Absence-of-errors fallacy
c) Pesticide fallacy
d) Testing is context dependent

36) With the right mindset developers are able to test their own what? Show Answer

The correct Answer is 'C'
a) Design documents
b) Requirements
c) Software
d) Test cases

37) Component testing is often used in testing all but which of the following? Show Answer

The correct Answer is 'B'
a) Regression testing
b) Structural-behavior
c) Robustness testing
d) Structural testing

38) Which tests are a strong candidate for automation? Show Answer

The correct Answer is 'B'
a) Performance tests
b) Regression tests
c) Load tests
 d) Usability tests

39) The outline of a test summary report is provided in which standard? Show Answer

The correct Answer is 'C'
a) ISO Std 989-1998
b) ISO Std 829-1998
c) IEEE Std 829-1999
d) IEEE Std T4S-V9B

40) All of the following are product risks except what? Show Answer

The correct Answer is 'A'
a) Contractual issues
b) Failure-prone software delivered
c) Software that does not perform its intended functions
d) Poor software characteristics
