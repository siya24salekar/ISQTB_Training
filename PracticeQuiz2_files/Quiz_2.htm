ISEB / ISTQB Foundation in Software Testing Training Course

 

Practice Quiz 2

1) Software that does not work correctly can cause all but which of the following? Show Answer

The correct Answer is 'C'
a) Decrease of profits
b) Business reputation
c) Increase of sales
d) Injury or death

2) What can contribute to the quality of the software system? Show Answer

The correct Answer is 'A'
a) Testing
b) Repetition
c) Upgrades
d) Patching

3) Testing activities include all but which of the following? Show Answer

The correct Answer is 'C'
a) Planning and control
 b) Choosing test conditions
c) Fabricating results
d) Designing and executing test cases

4) Which of the following is not a testing principle? Show Answer

The correct Answer is 'C'
a) Exhaustive testing is impossible
b) Absence-of-errors fallacy
c) Pesticide parody
d) Testing is context dependent

5) The fundamental test process consists of all but which following activities? Show Answer

The correct Answer is 'C'
a) Planning and control
 b) Analysis and design
c) Test opening activities
d) Implementation and execution

6) Designing and prioritizing high level test cases is associated with what? Show Answer

The correct Answer is 'C'
a) Evaluating Exit Criteria and Reporting
b) Test Closure Activities
 c) Test Analysis and Design
d) Test Implementation and Execution

7) With the right mindset developers are able to test their own what? Show Answer

The correct Answer is 'B'
a) Design documents
b) Documentation
c) Code
d) Test cases

8) Looking for failures in a system requires what? Show Answer

The correct Answer is 'B'
a) Repeatability
b) Curiosity
c) Design documents
d) White-box testing knowledge

9) What found and fixed during testing will save time and money later? Show Answer

The correct Answer is 'C'
a) Testware
b) Process docs
c) Defects
d) Hourly rates

10) Which of the following is not a valid software work product? Show Answer

The correct Answer is 'B'
a) Business scenarios or use cases
b) Project plans
c) Requirements specifications
d) Design documents and code

11) Which of the following is not a valid Iterative-incremental development model? Show Answer

The correct Answer is 'C'
a) Prototyping
b) Rational Unified Process (RUP)
c) V-Model
d) Agility development models

12) Component testing is often used in testing all but which of the following? Show Answer

The correct Answer is 'A'
a) Regression testing
b) Resource-behavior
c) Robustness testing
d) Structural testing

13) System integration testing may test the interactions between different what? Show Answer

The correct Answer is 'C'
a) Hardware and Software
b) Hardware and Documentation
c) Requirements and Hardware
d) Developers and Software

14) Testing of specific non-functional characteristics may be included in what? Show Answer

The correct Answer is 'C'
a) Requirements Analysis
 b) Regression testing
c) Integration testing
d) Acceptance testing

15) Which of the following is not part of the test basis for Acceptance testing? Show Answer

The correct Answer is 'C'
a) System requirements
b) Use cases
c) Test processes
d) Business processes

16) Operational Acceptance testing includes all but which of the following? Show Answer

The correct Answer is 'C'
a) Disaster recovery
b) Maintenance tasks
c) Developing of backup/restore functionality
d) Data load and migration tasks

17) The ______ are “what” the system does. Show Answer

The correct Answer is 'B'
a) Components
b) Functions
c) Systems
d) Results

18) Which tests are a strong candidate for automation? Show Answer

The correct Answer is 'B'
a) Performance tests
b) Regression tests
c) Load tests
 d) Usability tests

19) Any software work product can be what? Show Answer

The correct Answer is 'C'
a) Deployed
b) Developed
c) Reviewed
d) Tested

20) A typical formal review will include all but which of the following roles? Show Answer

The correct Answer is 'B'
a) Scribe
b) Developer
c) Moderator
d) Author

21) Which of the following is not a characteristic of an Inspection review? Show Answer

The correct Answer is 'C'
a) Usually conducted as a peer examination
b) Defined roles
c) Pre-meeting preparation by reviewers
d) Includes metrics gathering

22) Which of the following is not a typical defect discovered by static analysis? Show Answer

The correct Answer is 'D'
a) Inconsistent interfaces between modules and components
b) Variables that are not used or are improperly declared
c) Referencing a variable with an undefined value
d) Unreachable (undead) code

23) What is based on the analysis of the structure of the component or system? Show Answer

The correct Answer is 'D'
a) Acceptance testing
b) Functional testing
c) Black-box testing
d) White-box testing

24) What is another term used in software development for True or False? Show Answer

The correct Answer is 'C'
a) On or Off
b) Yes or No
c) Boolean
d) Up or Down

25) What are very useful for designing acceptance tests with customer/user participation? Show Answer

The correct Answer is 'C'
a) A decision table
b) Boundary value analysis
c) A use case
 d) A test procedure

26) Which testing is where tests are derived from the tester’s skill and intuition? Show Answer

The correct Answer is 'C'
a) Decision testing
b) Boundary value analysis
c) Experienced-based testing
d) Functional testing

27) Which of the following is not a factor in choosing a test technique? Show Answer

The correct Answer is 'A'
a) Test estimate
b) Regulatory standards
c) Customer or contractual requirements
d) Level of risk

28) Which of the following is not a valid option for independent testing? Show Answer

The correct Answer is 'B'
a) No independent testers; developers test their own code
b) Independent developers from the business organization or user community
c) Independent testers within the development teams
d) Independent testers outsourced or external to the organization

29) Which of the following is not a typical test leader task? Show Answer

The correct Answer is 'C'
a) Decide what should be automated, to what degree, and how
b) Decide about the implementation of the test environment
c) Write the test cases for the project
d) Write test summary reports based on the information gathered during testing

30) Typical entry criteria may cover all but which of the following? Show Answer

The correct Answer is 'B'
a) Test environment availability and readiness
b) Testable code testability
 c) Test tool readiness in the test environment
 d) Test data availability

31) The testing effort may depend on all but which of the following factors? Show Answer

The correct Answer is 'A'
a) Characteristics of the users
b) Characteristics of the product
c) Characteristics of the development process
d) The outcome of testing

32) The outline of a test summary report is given in which standard? Show Answer

The correct Answer is 'A'
a) IEEE Std 829-1998
b) ISO Std 829-1998
c) IEEE Std 829-1999
d) IEEE Std T4S-V9B

33) Which of the following is not a technical issues relating to project risk? Show Answer

The correct Answer is 'D'
a) Problems in defining the right requirements
b) Test environment not ready on time
c) Low quality of the design
d) The extent to which requirements have not been reviewed

34) What is used to reduce the risk of an adverse effect occurring? Show Answer

The correct Answer is 'D'
a) Releasing
b) Reviewing
c) Hiring
d) Testing

35) Tool support for testing is relevant to all but which of the following? Show Answer

The correct Answer is 'C'
a) Automating repetitive tasks
b) Test design
c) 35 billion US
 d) Test reporting

36) Which tools help find defects prior to dynamic testing? Show Answer

The correct Answer is 'C'
a) Review Tools
b) Modeling Tools
c) Static Analysis Tools
d) Test Management Tools

37) Which of the following is not a potential benefits of using tools? Show Answer

The correct Answer is 'B'
a) Repetitive work is reduced
b) Ease of access to information about developers or developing
c) Greater consistency and repeatability
d) Objective assessment

38) Which of the following is not a risk of using a tool? Show Answer

The correct Answer is 'B'
a) Unrealistic expectations for the tool
b) Risk of cost increase of open-source / free tool project
c) Underestimating the effort required to maintain the test assets
d) Over-reliance on the tool

39) Which of the following is not a consideration in selecting a tool for an organization? Show Answer

The correct Answer is 'C'
a) Evaluation against clear requirements
b) Evaluation of training needs
c) Ensuring an organizational maturity
d) Estimation of a cost-effect ratio

40) Which tools applied to source code can enforce coding standards? Show Answer

The correct Answer is 'A'
a) Static Analysis Tools
 b) Review Tools
c) Modeling Tools
 d) Test Management Tools
