ISEB / ISTQB Foundation in Software Testing Training Course

 

Practice Quiz 3

1) Which of the following produces a defect? Show Answer

The correct Answer is 'C'
a) A bug
b) A defect
c) An error
d) A fault

2) A common perception of testing is that it only consists of what? Show Answer

The correct Answer is 'A'
a) Running tests
b) Debugging
c) Developing
d) Regression

3) Which of the following is not a testing principle? Show Answer

The correct Answer is 'C'
a) Pesticide paradox
b) Exhaustive testing is impossible
c) Developing is context dependent
d) Absence-of-errors fallacy

4) The fundamental test process consists of all but which following activities? Show Answer

The correct Answer is 'C'
a) Analysis and design
b) Implementation and execution
c) Planning and reviews
d) Test closure activities

5) Reviewing the test basis is associated with what? Show Answer

The correct Answer is 'C'
a) Evaluating Exit Criteria and Reporting
b) Test Closure Activities
 c) Test Analysis and Design
d) Test Implementation and Execution

6) The mindset to be used while testing is different from that used while what? Show Answer

The correct Answer is 'C'
a) Black-box testing
b) Designing test cases
c) Developing
d) White-box testing

7) What is very constructive in the management of product risks? Show Answer

The correct Answer is 'B'
a) Design reviews
b) Testing
c) Customer support
d) Developer meetings

8) Looking for failures in a system requires all but which of the following? Show Answer

The correct Answer is 'B'
a) Curiosity
b) A critical personality
c) Professional pessimism
d) Attention to detail

9) A common V-model uses all but which of the following test levels? Show Answer

The correct Answer is 'C'
a) Component testing
b) Integration testing
c) Regression testing
d) System testing

10) Which of the following is not a valid Iterative-incremental development model? Show Answer

The correct Answer is 'B'
a) Prototyping
b) Waterfall mode
c) Rational Unified Process (RUP)
d) Agile development models

11) Component testing is often used to detect which of the following issues? Show Answer

The correct Answer is 'C'
a) Project Scheduling
b) Help Functionality
c) Typos
d) Memory leaks

12) System integration testing may be done after when? Show Answer

The correct Answer is 'A'
a) System testing
b) White-box testing
c) Black-box testing
d) Acceptance testing

13) In which testing, should the test environment should correspond to the production environment? Show Answer

The correct Answer is 'C'
a) Regression testing
b) Functional testing
c) System testing
 d) User acceptance testing

14) Which testing is the acceptance of the system by the system administrators? Show Answer

The correct Answer is 'C'
a) Alpha testing
b) Contract and Regulation Acceptance testing
c) Operational Acceptance testing
d) User Acceptance testing

15) Operational Acceptance testing includes all but which of the following? Show Answer

The correct Answer is 'C'
a) Disaster recovery
b) Data load and migration tasks
c) Regression planning
d) Testing of backup/restore functionality

16) What is termed; the testing of “how” the system works? Show Answer

The correct Answer is 'C'
a) Functional testing
b) Confirmation testing
c) Non-functional testing
d) Acceptance testing

17) The planning of releases in advance is crucial for successful what? Show Answer

The correct Answer is 'B'
a) Load testing
b) Maintenance testing
c) Usability testing
d) Confirmation testing

18) Compared to dynamic testing, static techniques find what? Show Answer

The correct Answer is 'B'
a) Amounts of failures
b) Causes of failures
c) Failures
d) Omissions in the test plan

19) Who is the person who leads the review of the document or set of document? Show Answer

The correct Answer is 'C'
a) An Author
b) A Scribe
c) A Moderator
d) A Manager

20) Which of the following is not a characteristic of an Walkthrough review? Show Answer

The correct Answer is 'B'
a) Ideally led by trained moderator
b) Inexpensive way to get some benefit
c) Pre-meeting preparation by reviewers
d) Optional use of checklists

21) Which of the following is not a characteristic of an Inspection review? Show Answer

The correct Answer is 'C'
a) Pre-meeting preparation
 b) Inspection report including list of findings
 c) May be performed as a peer review without management participation
d) Formal follow-up process

22) During test analysis, which documentation is analyzed? Show Answer

The correct Answer is 'D'
a) Test plan
b) Test cases
c) Project plan
d) Test basis

23) A boundary value for a valid partition is termed what? Show Answer

The correct Answer is 'D'
a) Invalid boundary value
 b) Valid partition value
c) Invalid partition value
d) Valid boundary value

24) Use cases may be described at which level? Show Answer

The correct Answer is 'C'
a) All levels
b) Structural testing
c) The abstract level
d) Black-box testing

25) Decision coverage is related to what? Show Answer

The correct Answer is 'C'
a) Decision testing
b) Boundary value analysis
c) Branch testing
 d) Statement testing

26) Which of the following is not a factor in choosing a test technique? Show Answer

The correct Answer is 'C'
a) Regulatory standards
 b) Customer or contractual requirements
c) Testing life-cycle
d) Level of risk

27) Which of the following is not a valid option for independent testing? Show Answer

The correct Answer is 'A'
a) Independent testers; testers test their own code
b) Independent testers within the development teams
c) Independent testers outsourced or external to the organization
d) Independent testers from the business organization or user community

28) Which of the following is not a typical tester task? Show Answer

The correct Answer is 'B'
a) Create test specifications
b) Review and approve test plans
c) Prepare and acquire test data
d) Measure performance of components and systems

29) Typically exit criteria may cover all but which of the following? Show Answer

The correct Answer is 'C'
a) Thoroughness measures, such as coverage of code, functionality or risk
b) Estimates of defect density or reliability measures
c) Schedules such as those based on time to complete requirements reviews
d) Residual risks, such as defects not fixed

30) The selected approach depends on all but which of the following? Show Answer

The correct Answer is 'B'
a) The technology
b) Health and safety
c) The nature of the system
d) The test objectives

31) Metrics should be collected during and at the end of what? Show Answer

The correct Answer is 'A'
a) A test level
b) A project
c) A week
d) A month

32) All but which of the following is related to project risk? Show Answer

The correct Answer is 'A'
a) Poor data integrity and quality
b) Organizational factors
c) Technical issues
d) Supplier issues

33) All of the following are product risks except what? Show Answer

The correct Answer is 'D'
a) Failure-prone software delivered
b) Software that does not perform its intended functions
c) Poor software characteristics
d) Skill, training and staff shortages

34) Which type of risks are a special type of risk to the success of a project? Show Answer

The correct Answer is 'D'
a) Acceptable risks
b) Unacceptable risks
c) Project risks
 d) Product risks

35) In a risk-based approach, the identified risks can be used to all but which of the following? Show Answer

The correct Answer is 'C'
a) Determine the test techniques to be employed
b) Determine the extent of testing to be carried out
c) Determine whether any developer activities could be employed to reduce risk
d) Prioritize testing in an attempt to find the critical defects as early as possible

36) Which of the following are not Management tools? Show Answer

The correct Answer is 'C'
a) Requirements Management Tools
b) Incident Management Tools
c) Review Tools
d) Configuration Management Tools

37) Which tools report on how a system behaves under a variety of simulated conditions? Show Answer

The correct Answer is 'B'
a) Test Management Tools
 b) Performance Testing/Load Testing/Stress Testing Tools
c) Static Analysis Tools
d) Dynamic Analysis Tools

38) A __________ testing approach separates out the test inputs. Show Answer

The correct Answer is 'B'
a) Test-driven
b) Data-driven
c) Developer-driven
d) Requirement-driven

39) Success factors for the deployment of the tool include all but which of the following? Show Answer

The correct Answer is 'C'
a) Monitoring tool use and benefits
b) Gathering lessons learned from all teams
c) Requiring ongoing support for the test team for a given tool
d) Defining usage guidelines

40) Which of the following does not relate to introducing a pilot project? Show Answer

The correct Answer is 'A'
a) Decide on ad-hoc ways of using the tool
b) Learn more detail about the tool
c) Evaluate how the tool fits with existing processes
d) Assess whether the benefits will be achieved at reasonable cost
