ISEB / ISTQB Foundation in Software Testing Training Course

 

Practice Quiz 4

1) An error can produce Which of the following? Show Answer

The correct Answer is 'C'
a) A mistake
b) A defect
c) A fault
d) An issue

2) Testing can have all but which of the following objectives? Show Answer

The correct Answer is 'A'
a) Creating defects
b) Finding defects
c) Gaining confidence
d) Preventing defects

3) Testing everything (all combinations of inputs and preconditions) is not what? Show Answer

The correct Answer is 'C'
a) Desirable
b) Allowed
c) Feasible
d) Always permitted

4) The fundamental test process consists of all but which following activities? Show Answer

The correct Answer is 'C'
a) Implementation and execution
b) Test closure activities
c) Static Analysis and design
d) Planning and control

5) Evaluating testability of the test basis is associated with what? Show Answer

The correct Answer is 'C'
a) Evaluating Exit Criteria and Reporting
b) Test Closure Activities
 c) Test Analysis and Design
d) Test Implementation and Execution

6) Testing independence is not a replacement for what? Show Answer

The correct Answer is 'C'
a) Permanent employees
b) Ad-hoc testing
c) Familiarity
d) Acceptance testing

7) It is important to clearly state the objectives of what? Show Answer

The correct Answer is 'B'
a) The Design documentation
 b) The Testing
c) The Test Process
d) The Test Manager

8) Looking for failures in a system requires all but which of the following? Show Answer

The correct Answer is 'B'
a) Curiosity
b) A Critical approach
c) Attention to detail
d) Error guessing

9) Test activities are related to which activities? Show Answer

The correct Answer is 'C'
a) Software support
b) Software reviewing
c) Software development
d) Software production

10) Which of the following is not a valid Iterative-incremental development model? Show Answer

The correct Answer is 'B'
a) Prototyping
b) Reactive Application Development (RAD)
c) Rational Unified Process (RUP)
d) Agility development models

11) Component testing is also known as all, but which of the following? Show Answer

The correct Answer is 'C'
a) Unit testing
b) Module testing
c) Developer testing
d) Program testing

12) Systematic integration strategies may be based on the system what? Show Answer

The correct Answer is 'A'
a) Architecture
b) Requirements
c) Plan
d) Test plan

13) Which of the following is not part of the test basis for Acceptance testing? Show Answer

The correct Answer is 'C'
a) Risk analysis reports
 b) Use cases
c) Development processes
d) Business processes

14) Which testing typically verifies the fitness for use of the system by business users? Show Answer

The correct Answer is 'C'
a) Regression testing
b) Functional testing
c) User Acceptance testing
d) System testing

15) Which testing is performed against a contract’s acceptance criteria? Show Answer

The correct Answer is 'C'
a) Alpha testing
b) Operational Acceptance testing
c) Contract and Regulation Acceptance testing
d) User Acceptance testing

16) Functional testing is considered to be what? Show Answer

The correct Answer is 'C'
a) Un-planned
b) White-box
c) Black-box
d) Ad-hoc

17) Maintenance testing for migration should include what? Show Answer

The correct Answer is 'B'
a) Usability tests
b) Operational tests
c) Confirmation tests
d) Conversion tests

18) Reviews, static analysis and dynamic testing have the same what? Show Answer

The correct Answer is 'B'
a) Design
b) Objective
c) Requirements
d) Test objects

19) Which of the following is not a valid formal review activity? Show Answer

The correct Answer is 'C'
a) Noting potential defects
 b) Follow-up
c) Examination/evaluation/recording of review minutes
d) Individual preparation

20) Who documents all the issues, problems found during a review meeting? Show Answer

The correct Answer is 'B'
a) A Moderator
b) A Scribe
c) A Manager
d) An Author

21) Which of the following is not a characteristic of an Technical review? Show Answer

The correct Answer is 'C'
a) May be performed as a peer review
b) Pre-meeting preparation by reviewers
c) May vary in practice from quite informal to very formal
d) Optional use of checklists

22) Which of the following is not a characteristic of an Inspection review? Show Answer

The correct Answer is 'D'
a) Pre-meeting preparation
b) Inspection report including list of findings
c) Formal follow-up process
 d) Results may be documented

23) The objective of what is to find defects in software source code and software models? Show Answer

The correct Answer is 'D'
a) Test case reviews
b) Black-box testing
c) White-box testing
d) Static analysis

24) What is performed without actually executing the software being examined by the tool? Show Answer

The correct Answer is 'C'
a) Test case reviews
b) Black-box testing
c) Static analysis
 d) White-box testing

25) Which of the following is not a typical defect discovered by static analysis? Show Answer

The correct Answer is 'C'
a) Inconsistent interfaces between modules and components
b) Variables that are not used or are improperly declared
c) Overly complicated design documents
d) Referencing a variable with an undefined value

26) Black-box and white-box testing also draws upon the experience of all but which of the following? Show Answer

The correct Answer is 'C'
a) Developers
b) Testers
c) Germany
d) Users

27) What describes interactions between actors, including users and the system? Show Answer

The correct Answer is 'A'
a) A use case
b) A test case
c) A test plan
d) A test procedure

28) Which of the following is not a factor in choosing a test technique? Show Answer

The correct Answer is 'B'
a) Regulatory standards
b) knowledge of the customers
c) Customer requirements
 d) Contractual requirements

29) Development staff may participate in testing, especially at which level? Show Answer

The correct Answer is 'C'
a) Higher
b) Functional
c) Lower
d) Acceptance

30) Which of the following is not a typical test leader task? Show Answer

The correct Answer is 'B'
a) Decide what should be automated, to what degree, and how
b) Coordinate the test strategy with the end users
c) Decide about the implementation of the test environment
d) Write test summary reports based on the information gathered during testing

31) What can be used to define when to start testing? Show Answer

The correct Answer is 'A'
a) Entry criteria
b) Exit criteria
c) Requirements
d) Test Specification

32) Once what is estimated, resources can be identified and a schedule can be drawn up? Show Answer

The correct Answer is 'A'
a) Test effort
b) Test cases
c) Test specification
d) Test results

33) Metrics can be used to assess all but which of the following? Show Answer

The correct Answer is 'D'
a) The adequacy of the test objectives for that test level
b) The adequacy of the test approaches taken
c) The effectiveness of the testing with respect to the objectives
d) The adequacy of the test cases used

34) Which of the following is not a technical issues relating to project risk? Show Answer

The correct Answer is 'D'
a) Problems in defining the right requirements
b) Test environment not ready on time
c) Late data conversion
d) Low quality of the test team

35) In a risk-based approach, the identified risks can be used to all but which of the following? Show Answer

The correct Answer is 'C'
a) Determine the extent of testing to be carried out
b) Prioritize testing in an attempt to find the critical defects as early as possible
c) Determine the test results to be documented
d) Determine whether any non-testing activities could be employed to reduce risk

36) The structure of an incident report is covered in which standard? Show Answer

The correct Answer is 'C'
a) ISO Std 829-1998
b) ISO Std V9B-T4S
c) IEEE Std 829-1998
d) IEEE Std 829-1999

37) Which of the following is not a potential benefits of using tools? Show Answer

The correct Answer is 'B'
a) Greater consistency and repeatability
b) Repetitive work is increased
c) Objective assessment
 d) Ease of access to information about tests or testing

38) Which tools execute test objects using automated test scripts? Show Answer

The correct Answer is 'B'
a) Review Tools
b) Test Execution Tools
c) Modeling Tools
d) Test Management Tools

39) Which of the following does not relate to introducing a pilot project? Show Answer

The correct Answer is 'C'
a) Learn more detail about the tool
b) Evaluate how the tool fits with existing processes
c) Assess whether the benefits will be achieved at a great cost
d) Decide on standard ways of using the tool

40) Which of the following does not relate to introducing a pilot project? Show Answer

The correct Answer is 'A'
a) Evaluate how the tool fits with future processes
b) Learn more detail about the tool
c) Decide on standard ways of using the tool
 d) Assess whether the benefits will be achieved at reasonable cost
